Project Overview

The goal of the project is to optimize the call-to-action (CTA) strategies used by Financial Services to maximize revenue per user. I plan on approaching this project by analyzing user interactions with various CTA combinations to identify which ones drive the highest engagement and conversions (which in turn will lead to increased revenue). I will also try understanding different user behaviors to help predict the optimal call-to-action (CTA). These processes will help understand the most important factors in generating revenue

Lastly, I grouped by revenue* to get the total revenue per combination*

# Function to categorize users by estimated annual income
def income_categories(user_estimated_income):
    if user_estimated_income < 50000:
        return 'Low Class'
    elif 50000 <= user_estimated_income < 120000:
        return 'Middle Class'
    elif 120000 <= user_estimated_income < 350000:
        return 'High Class'
    else:
        return 'Upper Class'

# creating a new column using the income data

temp_financial_data['incomeGrouped'] = financial_data['estimatedAnnualIncome'].apply(income_categories)

PERFORMANCE COMMENTS

I am relieved at how great the model performed!

I decided to go with a RandomForestRegressor model which is an ensemble model. RandomForestRegressor unlike its base counterpart, DecisionTreeRegressor, trains multiple trees sequentially as opposed to one. This makes it way more efficient.

On top of that, I trained the model using GridSearch. GridSearch allows you to fine-tune your model using custom hyperparameters. It is important to keep the curse of dimensionality in mind. As a result, I decided to have few options per hyperparameter

The model can still be improved! I can retrain with new hyperparameters or train other models and compare the results. The assignment does not say anything about training more than one model, so I am a bit weary, nonetheless
